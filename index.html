Inspection Rules
& Rewards

Система наград и штрафов за проверку NPC: багаж, документы, лицо, татуировки, решение

Концепция

После проверки NPC система оценивает Inspection Rules — набор правил из трёх слоёв. Каждое сработавшее правило даёт награду или штраф. Все правила аддитивны.

Одно правило = условие (FGameplayTagRequirements: Require + Ignore) + награда.
Три слоя правил

Правила собираются из трёх источников и конкатенируются в один массив.

СлойГде задаётсяПример
Global Campaign Definition −25 за каждую неоткрытую сумку (всегда, для всех NPC)
Daily Day Definition (DayData) Сегодня повышенный штраф за контрабанду
Per-NPC NPC Spawn Preset (FNPCDefinition) +100 за верную оценку документов этого конкретного NPC

Все три слоя аддитивны. Нет механизма override — исключения делаются через Clue.Truth.Exempt.* теги (см. ниже).

Два типа правил
1

One-shot правило

Без Item.*/Bag.*/Tattoo.* в условии

Проверяется один раз по clue-тегам NPC.

Clue.Decision.Quality.Best → +300 CR
2

Per-entity правило

Есть Item.* / Bag.* / Tattoo.* в условии

Проверяется для каждой сущности. 3 совпадения = награда ×3.

Item.Type.Contraband Item.State.Detected → +150 CR

Система определяет тип автоматически: если условие содержит тег из Item.*, Bag.* или Tattoo.* — правило per-entity.

Порядок событий
01Взаимодействие с NPC
Диалог, сканер, документы → учатся Clue.Action.* и Clue.Assessment.* теги
02Решение
Игрок нажимает Permit / Refuse / Arrest
Учится Clue.Decision.Action.* — награды ещё не выдаются
03Свободное время
Предметы на столе — можно брать и класть обратно
04NPC уходит
Snapshot: per-item Kept vs Confiscated, bIsDetected
05Finalize Inspection
Inject Truth tags → derive Finding + Decision Quality → merge Global + Daily + Per-NPC rules → build per-entity containers → evaluate → grant rewards
Примеры правил
Per bag: открыл сумку
require Bag.State.Opened
+50 CR
per bag
Global rule. Per-entity: Bag.* тег → итерирует сумки.
Per item: обнаружил контрабанду
require Item.Type.Contraband Item.State.Detected
+150 CR
per item
Global rule. Item.Type.Contraband = всегда нелегален.
Per item: обнаружил violation (нарушение дня)
require Item.State.Violation Item.State.Detected
ignore Item.Type.Contraband
+100 CR
per item
Ignore Contraband — чтобы не давать двойной бонус.
Aggregate: была контрабанда, но игрок ничего не нашёл
require Clue.Truth.Cargo.HasContraband
ignore Clue.Finding.Cargo.ContrabandDetected
−75 CR
One-shot. Truth = контрабанда была. Finding отсутствует = не нашёл.
Docs: верная оценка документов
require Clue.Truth.Docs.HasViolation Clue.Assessment.Documents.Violation
+100 CR
Per-NPC rule (только на NPC с нарушением). Truth + Assessment совпали.
Docs: пропустил нарушение
require Clue.Truth.Docs.HasViolation
ignore Clue.Assessment.Documents.Violation
−70 CR
Per-NPC rule. Нарушение есть, игрок не пометил.
Per tattoo: просканировал тату
require Tattoo.State.Scanned
+25 CR
per tattoo
Global rule. Tattoo.* тег → итерирует татуировки.
Decision: лучшее решение
require Clue.Decision.Quality.Best
+300 CR
Global rule. Quality вычисляется из Truth.Decision.Best.* + фактического решения.
Exempt: послу разрешена контрабанда
require Clue.Truth.Cargo.HasContraband
ignore Clue.Finding.Cargo.ContrabandDetected Clue.Truth.Exempt.Contraband
−75 CR
Штраф за пропуск контрабанды. Если NPC — посол с Exempt тегом → правило НЕ сработает.
Подсказки
  • Правила из всех трёх слоёв (Global, Daily, Per-NPC) складываются аддитивно. Порядок не влияет на результат.
  • Правило без Item.* / Bag.* / Tattoo.* тегов — one-shot, проверяет только clue-теги.
  • Правило с entity-тегами — проверяется для каждой сущности (предмет, сумка, тату). Сколько совпало — столько наград.
  • Item.State.Violation = нелегален сегодня (daily rules). Item.Type.Contraband = нелегален всегда. Разные вещи, разные награды.
  • Clue.Truth.* — факты о NPC, задаются дизайнером в TruthTags на NPC definition. Инжектятся в clue-контейнер при evaluation.
  • Clue.Finding.* — вычисляются автоматически из состояния предметов. Используются для aggregate условий («контрабанда была, ничего не нашёл»).
  • Ignore = «если тег есть — правило НЕ срабатывает». Используйте для исключений и Exempt.
  • Clue.Decision.Quality.* вычисляется из Truth.Decision.*.* — дизайнер задаёт какое решение лучшее/худшее для каждого NPC.
